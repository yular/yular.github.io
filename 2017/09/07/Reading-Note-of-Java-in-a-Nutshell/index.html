Reading Note of Java in a Nutshell (Java 7 and 8)

Here is the summary of the book Java in a Nutshell (JDK 7 and 8 version).

JVM and Java Platform

JVM executes Java program in Java bytecode format, which is called class files - the ones with .class extension. Java bytecode is an intermediate code which is between source code and machine code.

Java platform uses just-in-time compilation to take the advantage of the fact that parts of Java program are called more frequently.

Two ways to run other languages on JVM:

  • Have a source code compiler producing class files.
  • Implement an interpreter and runtime in Java, and then interpret the source form of the language.

Software Construction

  • Problem Definition
  • Requirement Development
  • Construction Planning
  • Software Architecture or High Level Design
  • Detailed Design

  • Unit Testing
  • Integration Testing
  • Integration

Coding and Style

Singleton

  • Use a framework (usually dependency injection) to provide singleton automatically.

Lambda expressions:

  • A lambda expression is an anonymous collection of executable Java code, essentially a method body.
  • Syntax of lambda expressions is like: ( paramlist ) -> { statements }.
  • When a lambda expression is used as a value, it is automatically converted to a new object of the correct type for the variable being placed into. This autoconversion and type inference is essential to Java’s approach to lambda expression.

Naming Convention:

  • Class: Class names are nouns to represent objects.
  • Interface: Interface names are adjective if to provide additional information about the class, otherwise are nouns to work like abstract superclasses.

Label statements:
A label statement is simply a statement that has been given a name by prepending an identifer and a colon to it.

Generic type:
For wildcard, it is legal to write in following style: List<?> objects = new ArrayList(). List is subtype of List<?>. But List<?> is not a subtype of any List.

Reflection (Convert specific to general):

  • Only deal with cases immediately applicable to problem domain.
  • To use reflection: Class<?> clz = getSomeClassObject(); Object rcvz = clz.newInstance();

Datatype Overflow:
Integer arithmetic never overflow or underflow because numbers just wrap around.

Handling Exception

Exceptions are Throwable objects and that exceptions fall into two main categories, specified by the Error and Exception subclass. Any exception object that is an Error is unchecked.

Security

A safe program includes:

  • All objects start off in a legal state after creation.
  • Externally accessible methods transition objects between legal state.
  • External accessible methods must not return with object in an inconsistent state.
  • External accessible methods must reset object to a legal state before throwing (exceptions).

Use java.util.Arrays.equals to compare the content of two arrays.

Concurrency

Two collectors:

  • Parallel collectors, using multiple threads to perform collections.
  • Concurrent collectors, running at the same time as application threads are still running.

IO

  • Use Readers and Writers to do IO operation.
  • Use try-with-resources (TWR) in IO operation to take care of resources management. TWR uses a new interface called AutoCloseable.
    TWR style is like: try(InputStream is = new FileInputStream(“”)) { … }
  • Use File class for file operation and Path class for file system path operation.
  • For high-performance IO, use ByteBuffer.
  • For Async IO, use AsynchronousFileChannel for file IO, AsynchronousSocketChannel for client socket IO, and AsynchronousServerSocketChannel for asynchronous sockets accepting incoming connection.
  • For network, use java.net or javax.net packages.
  • For Async style, use Future-Based style, Callback-Based style and watchservice on a directory.
  • Use MappedByteBuffer to store a memory-mapped file into memory.

Note :

  • ByteBuffer and MappedByteBuffer are NIO buffers which are low-level abstraction for high-performance IO.

Time

Use Java 8 Date and Time API: java.time .

Garbage Collection

Concurrent Mark and Sweep (CMS):
Only used to collect the old generation-conjunction with a parallel collector that is to clean young generation.

G1:
Used in high throughput environment and allows users to specify pause goals, which is low-pause collector.

Linux Server Trouble Shooting Commands and Tools

Here is the list of common used commands and tools for Linux server trouble shooting.

First of all, here are some useful links:

  • http://www.brendangregg.com/usemethod.html
  • http://www.brendangregg.com/USEmethod/use-linux.html
  • https://developers.google.com/web/tools/chrome-devtools/
  • http://www.brendangregg.com/blog/2016-08-09/gdb-example-ncurses.html

And below there are some useful commands and tools for server trouble shooting:

Basic Commands

  • fg & bg & jobs
    Start to run processes in foreground and background

  • ulimit
    Set the amount of available resources that can be used by system, including shell and processes.

  • rsync
    Fast, versatile, remote or local filecopying tool.

Text Processing Commands

  • sed, awk, grep, egrep, fgrep

  • zgrep
    Search possibly compressed files for a regular expression.

  • sort, uniq
    Sort given arrays and keeps each element unique.

Observation Tool

  • ps
    Report a snapshot of current processes.

  • pstree
    Display processes hierarchy as a tree shape.

  • top
    Display live view of system statistics.

  • htop
    A improved version of top command.

  • history
    Show executed command history.

  • w
    Display users logged in status.

  • df (-i -h)
    Report file system disk space usage.

  • du (-i -h)
    Estimate file system disk space usage.

  • free
    Display system memory usage.

  • iftop
    Display network bandwith usage of an interface by host.

  • iotop
    I/O top command.

  • iostat
    Report CPU and I/O statistics of devices, partitions and network filesystems(NFS).

  • vmstat
    Report virtual memory statistics.

  • mpstat
    Report processors related statistics.

  • dstat
    Versatile tool for generating system resource statistics.

  • dmesg
    Print or control the kernel ring buffer.

  • syslog
    Print system log.

  • uptime
    Shows how long the system has been running.

  • strace
    Displays the name of each system call together with its arguments enclosed in a parenthesis and its return value to standard error.

  • pidstat / swapon / collect / stat

  • ltrace / ss / iptraf / snmapget / lldptool / blktrace / slabtop / proc fs

Benchmark Tool

  • dd / hdparm / fio

  • unixbench
    See this link.
    Test basic graphics including 2D and 3D performance and Multi-CPU systems are also handled.

  • lmbench / sysbench / perf bench

  • ab / wrk / jmeter / openssl

  • ping / hping3 / iperf / ttcp / traceroute / mtr /
    pchar

Tunning Tool

  • sysctl
    Configures kernel parameters at runtime.

  • sysfs
    A pseudo file system provided by the Linux kernel that exports information about various kernel subsystems, hardware devices, and associated device drivers from the kernel’s device model to user space through virtual files.

  • nice / renice / taskset / chcpu

  • tune2fs / ionice / hdparm / blockdev

  • ethtool
    Prints current settings of the specified device. The device name is usually rendered as ethX, with X referencing the number of the device. The default Ethernet device is usually rendered as eth0.

  • tc
    Show / manipulate traffic control settings.

Tracing/Network Tool

  • ping
    Send ICMP echo_request to network hosts.

  • dig
    DNS lookup command.

  • traceroute
    Print route packets trace to target network host.

  • ifconfig
    Get or set configuration of a network interface.

  • netstat (-lntp, -natp, -r)
    Print network connections, routing tables, interface statistics.

  • ncat
    Display TCP and UDP connections and listeners.

  • tcpdump
    Dump traffic on a network.

  • telnet
    User interface of TELNET protocol.

  • ftrace
    Trace context switches, measure the time it takes for a high-priority task to wake up, the length of time interrupts are disabled, or list all the kernel functions executed during a given period.

  • perf_events
    See this Wiki link.

  • systemtap
    Instrument running production Linux kernel-based operating systems dynamically to extract, filter and summarize data in order to enable diagnosis of complex performance or functional problems.

  • sysdig
    See this Github link.Sysdig is a simple tool for deep system visibility, with native support for containers.

Monitoring Statistics

  • sar
    System Activity Reporter. See this link.
    Collect, report, or save system activity information.t

  • grafana

  • nagios

  • zabbix

  • smokeping

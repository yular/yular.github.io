Thinking in Web Developer

Here is the summary of my own web development experience as a engineer. Currently I am working on a web service project using Java in the backend and Javascript in the frontend. In the future, I will try to use Python or C++ to in the backend development.

Learn the Terminology

We should learn the terminology used in Web development at first so that we can understand the document and know how to find the solution from search engine such Bing and Google.

Currently I think the terminology in the below list is basic but important. Please ignore the order.:

Understand the WorkFlow

Developers should understand the workflow of the web project that they are working on, the tool that they will use, and the protocol that they will use.
Here the tools include text editor such as sublime and vim, and IDE such as eclipse. Of course, other tools are also possible.
Here the protocol should include HTTPS and HTTP. Note that some companies or agencies may use self-designed protocols and developers should read the document about them. Protocols define the communication behavior between different agencies of this project and data format of it.

Learn How to Use Development Tools

This is very important. Developers should know how to use the development tools to create, compile (if applicable), and run (a) project(s). For example, given an IDE, developers should know how to use it to develop a Java project through creating a Java project, compiling the Java after coding is done and then running it in the IDE.
They should also know how to use tools to debug. For example, developers should know how to use browser to figure out what is going wrong with their project: is it the bug coming from frontend, backend or even from third party?

Learn the Syntax and Coding Standard of Programming Language

Developers should learn the syntax of the programming language that they will use in the project. Most of the time, bugs are from syntax errors and we should avoid those unnecessary bugs.
To make our code more readable and easy to debug, we should follow the coding standard of that programming language. The standard include the variable naming standard and function naming standard.

Improve Trouble Shooting Skills

Most of time, we will meet lots of “tiny” technical issues in web development, such as the disappearance of elements in web page. These bugs are easy to fix if we have known some tricky approaches to do that. So that is the reason why we should know how to effectively use search engine or just directly go to some specific websites to find the solution.

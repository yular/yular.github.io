Hackerrank Data Structure Problem Solution List III

Here is the summary of Hackerrank data structure problem solution list.
summary list

Below is the detailed explanation of the idea behind solution.

Maximum Element

Problem Link
Code Link
Data structure problem. Use two stacks. One is to save all the given elements that are not poped out and the other is to save the maximum value starting from the element at the bottom of the first stack.

Largest Rectangle

Problem Link
Code Link
Data structure problem. Add 0 to the end of given array. Starting from the head of array, push the index of every element into a stack. If the element value pointed by the index at the top of stack is lower than that of the current index, pop out that index and then calculate the area formed between this index and the new top index at the stack; and update the answer if it is smaller than the new area.

Simple Text Editor

Problem Link
Code Link
Data structure problem. Use an array to simulate a stack.

Poisonous Plants

Problem Link
Code Link
Data structure problem. Use two stacks to do the simulation for the problem.

Waiter

Problem Link
Code Link
Data structure problem. At first find all the prime needed. Then use stack to do the simulation for the problem.

Jesse and Cookies

Problem Link
Code Link
Basic problem. Use queue to simulate the problem.

Castle on the Grid

Problem Link
Code Link
BFS problem. Find the path from source to destination with least number of changing direction.

Down To Zero II

Problem Link
Code Link
Basic problem. Use BFS to simulate the process.

Truck Tour

Problem Link
Code Link
BFS problem. Use BFS to simulate the process.

Queries with Fixed Length

Problem Link
Code Link
Monotone problem. Elements inside the queue is in non-increasing order. When insert element into the queue, starting from the tail of queue, pop back elements that has smaller value than the current one; and then starting from the head of queue, pop out the elements that are out of required boundary. Now in every query, the value of element at the head of queue is answer.

QHEAP1

Problem Link
Code Link
Priority queue problem. Use two min priority queues, one is to save all the values that have been removed and the other is to save all given values. Before output the minimum value, pop out the top value of the two priority queues until their top values are not equal.

Find Median

Problem Link
Code Link
Priority queue problem. Use one max priority queue and one min priority queue. Both of them save half of the given elements. The rule is that given a new element, if its value is larger than the value of top element of max priority queue, push that element into min priority queue; otherwise, push it into max priority queue. Now if difference between the the total number of elements of these two queues are larger than one, rebalance the queues so that the difference is no larger than one. For each query, if the total number of elements of the two queues are odd, the answer is the value of top element of max priority queue; otherwise, it is the half of the sum of the top value of two priority queues.

Minimum Average Waiting Time

Problem Link
Code Link
Greedy problem. Sort the order based on this rule: if two orders have the same start time, the one cost less time come first; otherwise, the order with less time come first. Then use priority queue to go through the sorted orders one by one and each time put the remaining order that cost most time to the top of priority queue.

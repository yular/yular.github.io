Hackerrank / InterviewStreet Algorithm Problem Solution List IV

Here are the solutions of some of the algorithm problems post in Hackerrank / InterviewStreet algorithm domain. The explanation of solution may not be detailed but every one can view my code in github. This page will be updated in any time.

Here is the summary of solution list.

Sherlock and Array

Problem Link
Code Link
Basic problem. Set sum[i] as the sum of all elements from 0 index to i index. Then for every index i, check whether sum[i - 1] and sum[i + 1] - sum[i] are equal or not.

Playing with numbers

Problem Link
Code Link
Sort the given numbers in non-descending order; then for every query number, adding the current number to sum of all previous query number; after that, if the sum is negative, using binary search to find the lower bound of the absolute value of the sum only in the positive part of the array; if the sum is positiive, using binary search to find the upper bound of the negative value of the sum only in the negative part of the arrary. At last, calculate the absolute value of the sum of the lower/upper bound part adding the sum of query number and the sum of other parts adding the sum of query number and finally, calculate the absolute value of the sum of all these two parts.
先对给定数组进行排序。然后将所有已知的查询数进行求和。接下来用二分搜索，如果和是负数，在数组的正数部分找出和的绝对值的下界；若是正数，在数组的负数部分找出和的取反的上界。最后，将>不需要进行二分搜索，和需要进行二分搜索这两部分进行求和。不需要二分搜索和在二分搜索结果以外的部分是直接加上查询和然后求绝对值即可；二分搜索结果范围内的也是这样，但三部分是需要分别>求出各自的绝对值之后再相加，此时的和就是最终答案。
